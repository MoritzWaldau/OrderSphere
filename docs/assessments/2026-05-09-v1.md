## 3. What is good? (Strengths)

| Area | Strength | Why it matters |
|---|---|---|
| Architecture | Clean Clean-Architecture layering with correct dependency direction | Eases testing, extension, and demonstrates senior-level understanding |
| Domain | `Result<T>` primitive with XOR invariant (success **or** failure) | Functional error handling instead of exceptions for business logic |
| Domain | Rich entities: `Product.AddToStock()` / `RemoveFromStock()`, `CartItem.Increase()`/`Decrease()`, `Order` with a private `_items` list and `IReadOnlyCollection` | No anemic models — business rules live in the domain |
| Domain | Domain-specific `Error` classes (`ProductErrors`, `CartErrors`, `OrderErrors`) | Type-safe, reusable errors instead of magic strings |
| Application | Consistent CQRS pattern (`ICommand`/`IQuery` + handler interfaces) | Clear read/write separation, easy to extend |
| Application | `CheckoutCartCommandHandler` uses a transaction + `.AsTracking()` (new) + cart clearing after checkout | Correct EF behaviour: with the default `NoTracking`, `RemoveRange()` would silently fail |
| Application | `CheckoutCartCommandHandler` generates a correlation ID via `Guid.CreateVersion7()` and logs it | Traceability for errors in the checkout flow |
| Infrastructure | EF configurations with indexes (unique slug/SKU), decimal precision, max lengths | Production-grade database schema |
| Infrastructure | `AuditSaveChangesInterceptor` automatically sets `CreatedAt`/`UpdatedAt` | Consistent auditing without boilerplate |
| Infrastructure | `QueryTrackingBehavior.NoTracking` as the default | Performant for read-heavy workloads |
| Infrastructure | Soft delete via `IsDeleted` + `AuditableEntity` | No hard data loss |
| UI / design | MudBlazor with MUD palette tokens (`var(--mud-palette-*)`) instead of hardcoded colors | Dark mode support + theme switching without a refactor |
| UI / state | `CartService` with an event pattern (`OnCartUpdated`) | Reactive UI without a global state manager |
| Aspire | `AppHost.cs` with local PostgreSQL + Service Bus emulator + WaitFor | Reproducible local dev environment with a single `dotnet run` |
| Tooling | Centralized NuGet versioning via `Directory.Packages.props` | Prevents version drift between projects |
| Documentation | `.github/UNIT_TEST_PLAN.md` (new) | Structured test plan with xUnit/Moq/bUnit and CI workflow templates |

## 4. What can be improved? (Medium-severity issues)

| Area | Issue | File / path | Recommendation |
|---|---|---|---|
| Application | No input validation of commands/queries | all handlers in `src/OrderSphere.Application/Features/` | **FluentValidation** + a MediatR `ValidationBehavior` as a pipeline |
| Application | Manual DTO mapping in handlers | `src/OrderSphere.Application/Models/` + handlers | Introduce **Mapster** (lightweight) or **AutoMapper** |
| Application | Code duplication in `AddToCartCommandHandler` (two branches for Carts.Add vs. Carts.Update) | `src/OrderSphere.Application/Features/Cart/AddToCart/AddToCartCommandHandler.cs` | Unified flow with `FirstOrDefaultAsync` → create-or-update |
| Domain | The `Address` value object has public setters and no validation | `src/OrderSphere.Domain/ValueObjects/Address.cs` | Rebuild as an immutable `record` with constructor validation |
| Domain | The `Order` aggregate does not check that `ShippingAddress` is non-null and items > 0 | `src/OrderSphere.Domain/Entities/Order.cs` | Enforce invariants in the `Order.Create(...)` factory method |
| Infrastructure | `ServiceBusPublisher` without a retry policy / DLQ handling | `src/OrderSphere.Infrastructure/ServiceBus/ServiceBusPublisher.cs` | A **Polly** retry policy (exponential backoff) or Azure-native resilience |
| Infrastructure | `AuditSaveChangesInterceptor` sets `UpdatedAt` even on `Added` (redundant) | `src/OrderSphere.Infrastructure/Interceptors/AuditSaveChangesInterceptor.cs` | `if (entry.State == Modified) auditable.UpdatedAt = ...;` |
| UI | `Checkout.razor` (548 lines): too large, coupon logic (`WELCOME10`) hardcoded | `src/OrderSphere.UI/.../Components/Pages/Shopping/Checkout.razor` | Move coupon logic into `ApplyCouponCommand`, split the form into sub-components |
| UI | Inner classes in the Razor file | `Checkout.razor:527-549` | Extract into `Components/Pages/Shopping/Models/` |
| UI | `OrderSphereComponentBase` lifecycle logic inverted/redundant | `src/OrderSphere.UI/.../Configuration/OrderSphereComponentBase.cs:13-25` | Replace with a simple `OnAfterRenderAsync(firstRender)` |
| Domain | `AuditableEntity` lives under `Domain/Abstraction/`, belongs in `Domain/Primitives/` | `src/OrderSphere.Domain/` | Folder refactor for consistency |

### 10.2 Method
